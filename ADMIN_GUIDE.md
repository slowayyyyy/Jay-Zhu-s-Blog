# 博客内容管理后台使用说明

后台地址：`http://localhost:4321/admin/`

前台地址：`http://localhost:4321/`

## 一、本地打开网站和后台

在项目目录打开 PowerShell。只浏览前台时运行：

```powershell
npm run dev
```

需要使用中文内容管理后台时，运行：

```powershell
npm run admin
```

这个命令会同时保证博客开发服务已启动，并启动后台本地内容代理。使用后台期间不要关闭这个 PowerShell 窗口；结束后台编辑时按 `Ctrl + C` 即可停止内容代理，博客前台仍会在后台运行。

开发服务会在后台运行。可用下面的命令管理：

```powershell
npm run dev:status
npm run dev:logs
npm run dev:stop
```

服务启动后，在 Chrome 或 Edge 中打开后台地址。本地模式不需要 GitHub 登录，后台会直接读取当前博客文件夹。点击保存或发布后，内容会写入博客文件；如果前台页面已经打开，刷新一次即可看到最新内容。

## 二、后台可以管理什么

### 文章

在“文章”中可以新增、编辑和删除文章，并选择：

- 学习笔记
- 科研记录
- 生活手账
- 标签
- 是否在首页置顶
- 是否保存为草稿

正文编辑器支持标题、列表、链接、图片和 Markdown。草稿不会显示在前台。

### 每日打卡

在“每日打卡”中可以填写日期、标题、总结、记录项目、相关链接和标签。发布后会自动按日期倒序显示。

### 标签管理

在“标签管理”中统一新增、修改或删除标签。每个标签都是独立条目。先新建并发布标签，后台会自动刷新关系数据，然后在文章或每日打卡的“标签”输入框中搜索选择。

文章保存的是稳定的标签关联，而不是显示名称。因此修改已有标签名称后，使用该标签的旧文章、搜索页和标签列表都会自动显示新名称，不需要逐篇修改。

### 网站与个人资料

在“网站与个人资料”中可以修改：

- 博客名称和页头副标题
- 开场页“点亮每一盏灯”等文案
- 首页介绍和座右铭
- 头像
- 一句话自我介绍
- “关于”页面四个大类的中英文标题
- 研究兴趣
- 阅读、运动和其他爱好
- 当前在做什么
- GitHub、邮箱等联系方式
- 生活手账独立的页面介绍和侧栏文案
- 音乐播放器歌单和音乐文件

上传的头像和正文图片会保存到 `public/uploads/`，音乐文件会保存到 `public/audio/`。

## 三、检查修改结果

完成编辑后打开前台并刷新。正式发布前建议运行：

```powershell
npm run check
npm run build
```

两个命令都成功，说明内容格式正确并且网站可以构建。

## 四、网站上线后的后台登录

当前后台预设的 GitHub 仓库是：`slowayyyyy/Jay-Zhu-s-Blog`。

网站上传到这个仓库后，可在正式域名的 `/admin/` 打开后台。线上 GitHub 登录需要在部署阶段配置 OAuth；在完成这一步之前，先使用本地中文版后台发布内容。

在线后台发布时会直接向 GitHub 提交内容。Cloudflare Pages 检测到提交后会自动重新构建并更新网站。

线上后台保存后，前台不是立刻同步，通常会有 1 到 3 分钟延迟。这是因为当前架构是：

1. 后台保存
2. 提交到 GitHub
3. Cloudflare Pages 重新构建
4. 新版本上线

如果你想确认线上是否已经同步到最新仓库内容，可以在项目目录运行：

```powershell
npm run verify:live
```

这个命令会直接检查线上首页、文章页、生活手账、每日打卡、关于页、RSS、后台入口和阅读量接口。

如果最终 GitHub 仓库名称不同，需要同步修改 `public/admin/config.yml` 中的 `repo`。
